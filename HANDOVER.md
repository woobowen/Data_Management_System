# HANDOVER.md

## 1. 当前交接状态

- 项目已完成《大作业一》两阶段主体功能开发与文档整理。
- 后端/前端均可本地启动，自动化测试与构建可通过。
- 本文档用于后续协作者接手“问卷内美化与细节优化”。

## 2. 本地运行

后端（仓库根目录）：

```bash
npm install
npm run dev
```

前端（`frontend/`）：

```bash
npm install
npm run dev
```

依赖：MongoDB 可用（默认 `mongodb://127.0.0.1:27017/survey_system_v1`）。

## 3. 已完成功能范围

### 阶段一

- 账号注册/登录
- 问卷创建、编辑、发布、关闭
- 单选/多选/文本/数字题
- 动态跳转与提交校验
- 问卷统计

### 阶段二

- 题库 CRUD 与选题复用
- 题目共享（按用户名）
- 版本化编辑（新版本生成）
- 历史回退（最新版本回退到历史内容）
- 使用关系追踪（含历史数据兼容推断）
- 单题跨问卷统计（按题目聚合多问卷答案）
- 题库主列表仅展示当前版本
- 版本号与版本备注并行展示

## 4. 协作者建议接手范围（你提到的“美化和细节优化”）

1. 管理后台 UI 统一视觉风格（按钮、间距、卡片层级、颜色语义）。
2. 问卷填写页动效与交互细节（错误提示、过渡、空状态、加载状态）。
3. 题库页可用性优化（筛选、搜索、信息密度、提示文案微调）。
4. 不改动后端契约前提下优化前端提示与容错体验。

## 5. 改动边界（避免破坏已验收功能）

- 不要改动现有后端接口路径与响应结构（除非同步改文档与测试）。
- 不要改变版本链语义（“当前版本回退到历史内容”）。
- 保持提交端“后端重演路径”为最终裁决。
- 若涉及题目结构字段变更，需同步 `SYSTEM_AND_API_DESIGN.md` 与 `TEST_CASES.md`。

## 6. 快速验收清单

1. 登录后可进入问卷主页与题库管理页。
2. 题库可新建、编辑、共享、查看历史、回退、查看使用关系、查看跨问卷统计。
3. 主列表显示“第 N 版 + 版本备注”，不显示版本 ID。
4. 题库回退后主列表仍只显示当前生效版本。
5. 问卷填写与统计链路不受题库功能影响。
